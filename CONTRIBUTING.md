# Contributing to ChainVote

Thank you for your interest in contributing to ChainVote! We welcome contributions from everyone.

## 🤝 How to Contribute

### Reporting Bugs
Before creating bug reports, please check the [existing issues](https://github.com/yourusername/chainvote/issues) as you might find out that you don't need to create one. When creating a bug report, please include:

- Your operating system name and version
- Any details about your local setup that might be helpful in troubleshooting
- Detailed steps to reproduce the bug
- Screenshots if applicable

### Suggesting Enhancements
Enhancement suggestions are tracked as [GitHub issues](https://github.com/yourusername/chainvote/issues). Create an issue and use the "enhancement" label.

### Pull Requests
We actively welcome your pull requests:

1. Fork the repo and create your branch from `main`
2. If you've added code that should be tested, add tests
3. Ensure the test suite passes
4. Make sure your code lints
5. Issue that pull request!

## 🛠️ Development Setup

### Prerequisites
- Node.js 18+
- npm 8+

### Local Development

1. **Fork and clone the repository**
   ```bash
   git clone https://github.com/yourusername/chainvote.git
   cd chainvote
   ```

2. **Install dependencies**
   ```bash
   npm run install:all
   ```

3. **Setup development database**
   ```bash
   npm run setup
   ```

4. **Start development servers**
   ```bash
   npm run dev
   ```

### Code Style
- Use ESLint for linting (if configured)
- Follow existing code patterns
- Write clear, descriptive commit messages
- Use conventional commit format when possible

### Testing
- Write tests for new features
- Ensure existing tests pass
- Test manually in browser

## 📋 Development Guidelines

### Security
- Never commit sensitive information (API keys, passwords, etc.)
- Use environment variables for configuration
- Follow security best practices

### Code Quality
- Write clean, readable code
- Add comments for complex logic
- Use meaningful variable and function names
- Follow React and Node.js best practices

### Documentation
- Update documentation for new features
- Add JSDoc comments for public APIs
- Keep README.md up to date

## 🐛 Bug Reports

To encourage active collaboration, please follow these guidelines when reporting bugs:

1. **Use a clear and descriptive title** for the issue
2. **Describe the exact steps** needed to reproduce the behavior
3. **Provide specific examples** whenever possible
4. **Include screenshots** and error messages if relevant
5. **Check if the issue** has already been reported

## 💡 Feature Requests

We love feature requests! When suggesting a new feature:

1. **Search existing issues** to see if it's already been suggested
2. **Provide a clear description** of the feature
3. **Explain the use case** and why it would be valuable
4. **Include mockups or examples** if applicable
5. **Consider the impact** on existing functionality

## 🤔 Questions

If you have questions about the project:

1. **Check the README** and documentation first
2. **Search existing issues** and discussions
3. **Create a new issue** with the "question" label if needed

## 📞 Getting Help

- **GitHub Issues**: For bugs, features, and questions
- **Discord**: For community discussion and support
- **Email**: For direct communication
